There is a critical issue going on with the Nautilus application in Stratos DC. The production support team identified that the application is unable to connect to the database. After digging into the issue, the team found that mariadb service is down on the database server.



Look into the issue and fix the same.


# Investigation

First as usual we need to identify and login to DB server this time round it is different server from App server

## Investigation 

Apparently the instruction mentioned that the application was down. 

So I tried to start it 

`systemctl start mariadb.service`
